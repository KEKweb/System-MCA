**МСА (Многофункциональная система агентов)** — это суверенный, локальный программно-аппаратный комплекс для оркестрации больших языковых моделей (LLM), развернутый на **списанных графических ускорителях для майнинга (серия NVIDIA CMP)**.

В основе проекта лежит концепция **Hardware-Software Co-design** (совместное проектирование аппаратуры и ПО). Система успешно обходит искусственные аппаратные блокировки производителя (ограничения FP16 и тензорных ядер), обеспечивая производительность уровня Enterprise-серверов на бюджетном оборудовании.

## 🌟 Ключевые особенности и инновации

1. **Аппаратно-ориентированная оптимизация инференса:**
   - **Обход блокировки (DP4A Bypass):** На узлах NVIDIA CMP 90HX (Ampere) микрокод блокирует операции с плавающей точкой. Система принудительно использует целочисленное квантование `Q8_0` (INT8), переводя вычисления на свободные ALU-блоки через инструкции `DP4A`.
   - **Смягчение узкой шины (PCIe x1):** Для работы тяжелых моделей (Qwen 14B) через медленные майнинговые райзеры применяется строгий **Конвейерный параллелизм (Pipeline Parallelism)** (`--split-mode layer`). Это снижает задержки передачи скрытых состояний между картами до миллисекунд.

2. **Переключение контекста с нулевой задержкой (Late-Binding Role Injection):**
   - Для сохранения 100% эффективности Prefix Caching (кэширования промпта) в `llama.cpp`, Системный Промпт остается статичным. Инструкции конкретных агентов (Роутер, Критик, Эксперт) динамически внедряются в конец контекста (хвост запроса).

3. **Zero-Knowledge RAG (Абсолютная приватность):**
   - Фрагменты документов пользователя шифруются на стороне клиента (в браузере) по стандарту `AES-256-GCM` перед отправкой в векторную базу данных `Qdrant`. Сервер оперирует исключительно зашифрованными данными.

4. **Семантическая маршрутизация (Semantic Routing):**
   - Первичная маршрутизация простых запросов (определение интентов) выполняется без вызова LLM, за счет сравнения косинусного сходства векторов через модель `BGE-M3`. Это радикально экономит вычислительные ресурсы основного кластера.

## 🏗️ Архитектура кластера

Система использует гетерогенную топологию для распределения вычислительной нагрузки:
* **Узел А (Генерация и Интеллект):** Кластер NVIDIA CMP 90HX, выполняющий генерацию текста в INT8 формате с высокой пропускной способностью памяти.
* **Узел Б (Система Восприятия):** Кластер NVIDIA CMP 50HX, работающий в нативном FP16. Отвечает за распознавание документов (OCR `GOT-OCR-2.0`) и векторизацию (Embeddings `BGE-M3`).
* **Оркестратор:** Бэкенд на Python + `LangGraph`, управляющий машиной состояний и циклами верификации агентов (Эксперт -> Критик -> Рефакторинг).

## 📂 Структура проекта

* `app/agents/` — Узлы LangGraph, логика маршрутизации и определения состояния агентов.
* `app/services/` — Взаимодействие с RAG (Qdrant), OCR и веб-поиском (SearXNG).
* `app/crypto.py` — Логика клиентского шифрования (AES-256-GCM).
* `app/main.py` — Точка входа FastAPI (OpenAI-совместимый API).
* `static/` — Веб-интерфейс (Frontend) с поддержкой IndexedDB, Markdown и визуализации цепочек рассуждений (CoT).

## 🚀 Как запустить

> **Примечание:** Для полноценной работы требуется предварительно развернутая инфраструктура (Qdrant, SearXNG) и настроенный кластер инференса `llama.cpp` согласно спецификациям в документации.

1. Установите зависимости:
```bash
pip install -r requirements.txt
```

2. Запустите оркестратор:
```bash
uvicorn app.main:app --host 0.0.0.0 --port 8080
```
3. Откройте веб-интерфейс по адресу: `http://localhost:8080/static/index.html`
